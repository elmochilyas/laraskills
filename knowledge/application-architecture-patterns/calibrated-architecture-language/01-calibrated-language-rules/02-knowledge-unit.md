# Executive Summary

Architectural guidance should use calibrated language: prefer "default to", "usually", "when appropriate", "consider", and "avoid unless" over absolute terms like "always", "never", "every", or "all must". Reserve absolute language for security-critical rules, correctness invariants, and data integrity requirements where deviation always causes harm.
