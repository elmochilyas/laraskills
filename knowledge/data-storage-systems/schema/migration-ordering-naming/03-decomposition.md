# Decomposition: 1.6 Migration ordering and naming (YYYY_MM_DD_HHmmss)

## Topic Overview
Laravel executes migrations in lexicographic filename order. The `YYYY_MM_DD_HHmmss` prefix enforces deterministic sequencing. Naming conventions must encode the migration's intent — creation, alteration, index addition, or data change — and respect foreign key dependency ordering.

## Boundary Analysis
**In scope:** - **Timestamp prefix**: `2026_06_02_123456_create_posts_table.php` — generated by `date('Y_m_d_His')` when running `php artisan make:migration`.; - **Lexicographic sort**: Laravel sorts migration files by full filename string using PHP's `sort()`. `2026_01_01_000000` comes before `2026_01_01_000001`.; - **Batch tracking**: The `migrations` table stores the batch number per migration file. A single `php artisan migrate` command assigns the same batch number to all executed migrations.; - **FK ordering dependency**: If `create_books_table` references `authors` via foreign key, `create_authors_table` must have an earlier timestamp.; - **Name readability**: The portion after the timestamp communicates the migration's purpose. Following team conventions improves maintainability at scale..
