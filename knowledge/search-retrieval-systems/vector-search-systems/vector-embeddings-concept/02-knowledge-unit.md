## Executive Summary

Vector embeddings convert text (or images, audio, etc.) into numerical vector representations that capture semantic meaning. Similar concepts produce similar vectors, enabling semantic search. Embeddings are generated by neural network models (transformers) and stored in vector databases or indexes for similarity search.

## Core Concepts

- **Embedding Vector**: Fixed-size numerical array (e.g., 1536 dimensions for OpenAI text-embedding-3-small)
- **Semantic Proximity**: Similar vectors ˜ similar meaning
- **Dimensionality**: Higher dimensions capture more nuance but require more storage
- **Normalization**: Unit vectors ensure cosine similarity = dot product
- **Embedding Models**: OpenAI, Cohere, BGE, E5, FastEmbed
- **Use Cases**: Semantic search, clustering, classification, recommendation
