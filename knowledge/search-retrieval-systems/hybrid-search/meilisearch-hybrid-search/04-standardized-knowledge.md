## Overview

Meilisearch hybrid search combines keyword (full-text) and semantic (vector embedding) search results within a single query. Meilisearch generates auto-embeddings using built-in embedding models or accepts external embeddings, then fuses results using a built-in fusion algorithm (default RRF-like). The hybrid search feature makes Meilisearch one of the easiest engines to set up for mixed keyword + semantic search.

## Core Concepts

- **Auto-Embeddings**: Meilisearch can generate embeddings using built-in models (no external API needed).
- **External Embeddings**: You can provide your own embeddings generated by external models (OpenAI, Cohere).
- **Fusion Algorithm**: Meilisearch uses an internal fusion method to combine keyword and vector results.
- **Embedder Configuration**: Configure which model or external endpoint to use for embedding generation.
- **Hybrid Search API**: Search with `hybrid: { embedder: "default", semanticRatio: 0.5 }` parameter.

## When To Use

- Applications needing hybrid search with minimal setup
- Teams that want keyword + semantic search without managing separate engines
- Already using Meilisearch and want to add semantic capabilities
- Prototyping hybrid search before optimizing with a dedicated fusion approach

## When NOT To Use

- Fine-grained control over fusion parameters (Meilisearch abstracts these)
- Custom external re-rankers (cross-encoder) — Meilisearch doesn't support custom re-ranking
- Very large vector datasets — Meilisearch's vector search is newer than dedicated vector DBs
- When you need to use multiple embedding models for different document types

## Best Practices

1. **Start with auto-embeddings**: Built-in models work well for most use cases.
2. **Configure semanticRatio**: Balance between keyword (0) and semantic (1) results.
3. **Use external embeddings**: For higher quality, use OpenAI or Cohere embeddings via configuration.
4. **Test with your data**: Hybrid search effectiveness varies by content type.
5. **Monitor fusion balance**: Ensure neither keyword nor vector side dominates results.

## Architecture Guidelines

- Configure embedder in Meilisearch server settings (not in Scout config).
- For Scout integration, use the callback API to pass hybrid parameters.
- Meilisearch handles embedding generation during indexing (adds indexing time).
- Hybrid queries still benefit from all Meilisearch features (filtering, faceting, sorting).

## Performance Considerations

- Auto-embedding during indexing adds processing time (varies by model and document count).
- Hybrid query latency = max(keyword_latency, vector_latency) + fusion overhead.
- Embedding generation can be CPU/GPU intensive for large indexes.
- Built-in embedding models run on the Meilisearch server, consuming its resources.
