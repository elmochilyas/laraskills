# Skill: Learn from Real-World Laravel Integration Patterns

## Purpose
Study real-world Laravel API integration case studies to understand patterns, pitfalls, and best practices for common integration scenarios.

## When To Use
- Learning from production integration experiences
- Designing complex integration architectures
- Avoiding known pitfalls in API integration design
- Making architectural decisions for new integrations

## When NOT To Use
- Simple, well-documented integrations
- When direct vendor documentation is sufficient

## Prerequisites
- Basic understanding of Laravel integration patterns

## Workflow
1. Review case studies: payment gateways (Stripe, Paddle), CRM (Salesforce, HubSpot), communication (Twilio, Slack), AI (OpenAI, Anthropic)
2. Identify common patterns: service classes, webhook handling, retry strategies, error handling
3. Note anti-patterns: synchronous webhook processing, missing retry, hardcoded credentials
4. Extract reusable patterns for your integration
5. Document integration decisions and reasoning
6. Share patterns with team for consistency
7. Adapt patterns to specific integration requirements
8. Update case study knowledge as technology evolves

## Validation Checklist
- [ ] Case studies reviewed for relevant integration types
- [ ] Common patterns identified and extracted
- [ ] Anti-patterns documented to avoid
- [ ] Reusable patterns adapted to project needs
- [ ] Integration decisions documented with reasoning
- [ ] Patterns shared with team
