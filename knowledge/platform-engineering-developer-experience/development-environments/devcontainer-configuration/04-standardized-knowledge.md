## Overview

Devcontainer configuration provides a standardized, containerized development environment using VS Code's Dev Containers spec (`.devcontainer/devcontainer.json`). Defines Docker image, extensions, settings, port forwarding, and post-create commands. Sail generates devcontainer config via `php artisan sail:install --devcontainer`. Ensures identical PHP, Composer, Node, database, and service versions across the team. Works with GitHub Codespaces for cloud-based development.

## Core Concepts

- **devcontainer.json**: config file specifying Docker image, VS Code extensions, settings, forwarded ports, postCreateCommand
- **Dev Container**: Docker container configured as full dev environment with PHP, Composer, Node, DB, services
- **VS Code Extension Installation**: extensions installed automatically when opening project in container
- **Post-Create Commands**: scripts run after container creation (composer install, npm install, migration)
- **Port Forwarding**: container ports forwarded to host for browser access
- **GitHub Codespaces Integration**: same config works locally and in codespaces
- **Sail Generation**: `php artisan sail:install --devcontainer` creates devcontainer from Sail services

## When to Use

- Teams using VS Code wanting environment consistency
- GitHub Codespaces for cloud-based development
- Onboarding — new devs get fully configured environment with no manual setup
- Open-source projects for instant contributor setup

## When NOT to Use

- Teams not using VS Code (use Sail directly)
- Projects where Docker overhead is prohibitive (limited RAM)
- Solo projects where dev controls their own environment

## Best Practices (WHY)

- **Use Sail integration**: `sail:install --devcontainer` generates config consistent with Sail's services
- **PostCreateCommand for setup**: keep Dockerfile minimal (core tools); install project deps in postCreate
- **Extension standardization**: specify required VS Code extensions for consistent IDE tooling
- **Handle Codespace URLs**: APP_URL must include codespace hostname; use dynamic configuration
- **Use `.gitignore` for devcontainer**: not needed in production but safe to commit (no runtime impact)
- **Lifecycle hooks**: onCreateCommand (once), postCreateCommand (after create), postStartCommand (every start)

## Architecture Guidelines

- Generated by Sail's `--devcontainer` flag for consistency
- Docker Compose for multi-service setup (database, cache, mail)
- Keep environment variables in `.env` or `remoteEnv` — never hard-code secrets in devcontainer.json
- Pre-built image (Sail) for faster startup; custom Dockerfile for specific needs

## Performance Considerations

- First-time build: 5-15 minutes (image pull, deps)
- Bind mount filesystem overhead on macOS (use :cached or :delegated)
- RAM: 2-4GB for full Laravel devcontainer with services
- Startup: 10-30 seconds (stopped container)

## Security Considerations

- Devcontainer tools (Xdebug, dev extensions) should never run in production
- Secrets via VS Code remoteEnv or .env files, not hard-coded in devcontainer.json
- Codespace costs based on compute hours — configure machine type

## Common Mistakes

| Mistake | Description | Consequence | Better Approach |
|---------|-------------|-------------|-----------------|
| Not updating devcontainer for dep changes | New PHP extension needed | Container doesn't match project needs | Update Dockerfile when deps change |
| Hard-coding secrets in config | Keys in version control | Credential exposure | Use .env files |
| Ignoring Codespace URLs | APP_URL wrong in codespace | Broken redirects | Dynamic URL configuration |
| Building everything into image | Every dep change needs rebuild | Slow iteration | Install deps in postCreateCommand |

## Anti-Patterns

- **Maintaining devcontainer separately from Sail**: use Sail's generator to keep them in sync
- **Prod-like security in dev container**: dev containers have relaxed security intentionally

## Examples

```json
{
    "name": "Laravel Sail",
    "dockerComposeFile": "docker-compose.yml",
    "service": "laravel.test",
    "workspaceFolder": "/var/www/html",
    "forwardPorts": [80, 443, 3306, 8025],
    "postCreateCommand": "composer install && npm install && php artisan migrate"
}
```
