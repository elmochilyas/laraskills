# Decomposition: Stack Selection Guide

## Metadata
- **Domain:** Laravel Core Application Engineering
- **Subdomain:** Livewire / Inertia Basics
- **Knowledge Unit:** Stack Selection Guide
- **Difficulty Level:** Foundation

## Atomic Chunks

### Chunk 1: Livewire Characteristics
- **Topics:** Server-rendered HTML, minimal JS, real-time via AJAX
- **Key Content:** How Livewire works, PHP-centric development, Blade-based templates, no build step required
- **Learning Objectives:** Describe Livewire's architecture and identify the developer experience it provides

### Chunk 2: Inertia Characteristics
- **Topics:** Full JS frameworks (React/Vue/Svelte), client-side routing, SPA-like UX
- **Key Content:** How Inertia bridges Laravel with JS frontends, build tooling requirement, component architecture
- **Learning Objectives:** Describe Inertia's architecture and identify the developer experience it provides

### Chunk 3: Decision Criteria
- **Topics:** Team skills, UI complexity, SEO requirements, build tooling tolerance
- **Key Content:** Mapping project characteristics to stack choice: CRUD vs complex UI, JS expertise, SSR needs
- **Learning Objectives:** Evaluate project requirements against stack characteristics to make an informed selection

### Chunk 4: Migration Path Between Stacks
- **Topics:** Starting with Livewire and migrating to Inertia, co-existence strategies
- **Key Content:** Gradual migration from Blade/Livewire to Inertia, hybrid app patterns, risk mitigation
- **Learning Objectives:** Plan a gradual migration from one stack to another when requirements evolve
