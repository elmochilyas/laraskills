## Overview

Route names provide a stable, abstract reference to routes that decouples URL generation from the actual URI pattern. The `route()` helper generates URLs by route name, and named routes enable tests, views, and controllers to reference routes without hardcoding URLs.

Route names are defined via `->name('route.name')` on route definitions, automatically generated by `Route::resource()` (e.g., `photos.index`, `photos.store`), or prefixed via `Route::name('prefix.')` on groups. The framework maintains a `$nameList` hash table in the RouteCollection for O(1) named route lookup.

---

## Core Concepts

### named() Method
`Route::get('/users', ...)->name('users.index')` assigns a name. The name must be globally unique.

### Resource Route Names
`Route::resource('photos', ...)` automatically generates: `photos.index`, `photos.create`, `photos.store`, `photos.show`, `photos.edit`, `photos.update`, `photos.destroy`.

### Name Prefixes in Groups
`Route::name('admin.')->group(...)` prepends `admin.` to all route names within the group.

### route() Helper
`route('users.show', $user)` generates the full URL for the named route. Parameters are appended as URL segments or query parameters.

### action() Helper
`action([UserController::class, 'show'], $user)` generates the URL from the controller/action pair. May be slower than named routes due to action-based lookup.

---

## When To Use

- ALL routes — every route should have a name
- URL generation in views, emails, and controllers
- Redirect responses in controllers
- Test assertions that verify correct URLs
- Anywhere the route URL is referenced

---

## When NOT To Use

- Trivial redirect-only routes (though naming is still good practice)
- Routes that should not be publicly referenced (but name them anyway for internal use)

---

## Best Practices

### Name Every Route
Always call `->name()` on every explicit route definition.

**Why:** Named routes decouple URL references from URI patterns. Changing a URI requires only changing the route definition, not every reference. Tests using `route('name')` continue to work unchanged.

### Use Dot Notation Names
Use hierarchical dot notation: `resource.action` (e.g., `users.show`, `admin.users.index`).

**Why:** Dot notation mirrors the route hierarchy and the resource structure. It provides predictable naming that developers can guess without checking route files.

### Use Name Prefixes Matching URL Prefixes
When `Route::prefix('admin')` is used, also use `Route::name('admin.')`.

**Why:** Consistent prefixing makes route names predictable: URL `/admin/users` → route name `admin.users.index`. Developers can infer the route name from the URL.

### Prefer route() Over Hardcoded URLs
Never hardcode `/users/5` in views or controllers. Use `route('users.show', $user)`.

**Why:** Hardcoded URLs break when URIs change. Named routes are URI-agnostic.

## Architecture Guidelines

### Route Name Resolution
```
route('users.show', ['user' => 5])
  → RouteCollection::$nameList['users.show']
  → Route::uri() + parameter substitution
  → https://example.com/users/5
```

### Named Route Registration
```php
Route::get('/users/{user}', [UserController::class, 'show'])
    ->name('users.show');
```

### Resource Naming
```php
Route::resource('photos', PhotoController::class);
// Names: photos.index, photos.create, photos.store, photos.show
//        photos.edit, photos.update, photos.destroy
```

### Group Name Prefixing
```php
Route::prefix('admin')->name('admin.')->group(function () {
    Route::get('/users', [AdminController::class, 'index'])->name('users.index');
    // Full name: admin.users.index
});
```

## Performance Considerations

Named route lookup is O(1) via the `$nameList` hash table. The `route()` helper performance is dominated by URL parameter substitution, not name lookup. Named routes are included in route caching.

---

## Security Considerations

### Name Collision
Duplicate route names silently overwrite. The later registration wins. One route may be unreachable via `route()` while still matching via URI. Use unique, descriptive names to prevent collisions.

### Name Exposure
Route names appear in `route()` generated URLs only as URL segments (via parameters). The name itself is NOT exposed in the URL — only the URI defined in the route.

---

## Common Mistakes

### Not Naming Routes
Desc: Routes defined without `->name()`.
Cause: The route works without a name.
Consequence: Cannot generate URLs via `route()`, must hardcode URLs throughout the application.
Better: Always name every route.

### Duplicate Route Names
Desc: Two routes with the same name.
Cause: Copy-pasting route definitions without updating names.
Consequence: `route('name')` resolves to the later-registered route; the first is unreachable by name.
Better: Ensure globally unique route names.

### Omitting Name Prefix Separator
Desc: `Route::name('admin')` instead of `Route::name('admin.')`.
Cause: Forgetting the trailing dot.
Consequence: Route names become `adminusers.index` instead of `admin.users.index`.
Better: Always include the trailing `.` in name prefixes.

---

## Anti-Patterns

### Hardcoded URLs
Writing `/users/{{ $user->id }}/edit` in Blade templates instead of `route('users.edit', $user)`. Hardcoded URLs break when routes change and are not testable.

### action() for Named Routes
Using `action([Controller::class, 'method'])` for URL generation when the route has a name. `route('name')` is faster, simpler, and not coupled to controller class names.

## Examples

### Named Route Usage
```php
// Route definition
Route::get('/users/{user}', [UserController::class, 'show'])->name('users.show');

// URL generation
$url = route('users.show', $user); // https://example.com/users/5

// Redirect
return redirect()->route('users.index');

// Test assertion
$response->assertRedirect(route('users.index'));
```

### Resource with Custom Name
```php
Route::resource('photos', PhotoController::class)->names([
    'index' => 'gallery',
    'show' => 'gallery.show',
]);
```

## AI Agent Notes

### Important Decisions
- Route names must be globally unique — no two routes can share a name
- Resource routes auto-generate names: `resource.action`
- Name prefixes in groups concatenate with route names
- The `$nameList` hash table enables O(1) named route lookup

### Important Constraints
- `route()` requires the route name and all required parameters
- Missing route name throws `InvalidArgumentException` at call time
- Named routes are included in `route:cache`
- Duplicate names silently overwrite — first registration is lost

### Rules Generation Hints
- Enforce `->name()` on every route definition
- Enforce dot notation naming convention
- Enforce name prefixes matching URL prefixes in groups
- Ban hardcoded URLs in views and controllers
