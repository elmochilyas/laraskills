# Rules: Eloquent withSum/withAvg/withCount and Subquery Patterns

## Rule EA-01: Prefer withCount Over loadCount
`withCount` MUST be used when counts are needed for all query results. `loadCount` is only acceptable for conditional loading.

## Rule EA-02: Index Foreign Keys
Foreign key columns used in aggregate subqueries MUST be indexed. Without an index, correlated subqueries perform full table scans per parent row.

## Rule EA-03: Limit Selected Columns
When using aggregate methods, SELECT MUST be limited to only the columns needed using `select()`.

## Rule EA-04: Review Generated SQL
The SQL generated by aggregate methods MUST be reviewed during development to ensure index usage and query efficiency.

## Rule EA-05: Use Relationship Aliasing
Multiple aggregates on the same relationship MUST use aliasing: `withCount('orders as paid')`.

## Rule EA-06: Consider Pre-Computation for Large Data
Aggregates over datasets exceeding 1M rows MUST consider pre-computed counters or materialized views instead of query-time aggregation.

## Rule EA-07: Use whereExists for Existence Checks
Existence checks MUST use `whereExists()` or `whereHas()` (which generates WHERE EXISTS). `WHERE IN (SELECT ...)` patterns are less efficient.

## Rule EA-08: Test Aggregate Performance
Aggregate query performance MUST be tested with production-scale data volumes before deployment.

## Rule EA-09: Document Subquery Patterns
Complex subquery patterns used with `addSelect` MUST be documented with the equivalent SQL for debugging and review.

## Rule EA-10: No Aggregates on Lazy Collections
Aggregate methods MUST NOT be called on lazy-loaded relationships after the parent query has executed. Always use `with*` methods on the parent query.
