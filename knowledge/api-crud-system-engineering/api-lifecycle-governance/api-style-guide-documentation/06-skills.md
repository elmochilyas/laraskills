# Skill: Implement API Style Guide Documentation

## Purpose
Create and maintain an API style guide documenting conventions: naming, versioning, error format, pagination, authentication, headers, and response structure — serving as the single source of truth for API design.

## When To Use
- API design consistency
- Team onboarding
- External developer reference

## Inputs
- API conventions and decisions
- Architecture Decision Records

## Workflow
1. Create style guide document in project docs
2. Document: naming conventions (plural, kebab-case, resources vs actions)
3. Document: versioning strategy and URL structure
4. Document: authentication methods (Sanctum/Passport/API keys)
5. Document: error response format and codes
6. Document: pagination format (offset/cursor, metadata structure)
7. Document: filtering, sorting, include query parameters
8. Document: headers (rate limit, deprecation, security)
9. Document: response envelope structure
10. Review and update style guide quarterly

## Validation Checklist
- [ ] Style guide created and maintained
- [ ] Naming conventions documented
- [ ] Versioning strategy documented
- [ ] Authentication documented
- [ ] Error format documented
- [ ] Pagination documented
- [ ] Query parameters documented
- [ ] Headers documented
- [ ] Response structure documented
- [ ] Reviewed quarterly
