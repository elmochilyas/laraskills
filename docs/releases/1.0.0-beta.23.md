# LaraSkills 1.0.0-beta.23

**Release date:** 2026-06-23
**Type:** Feature release — Assistant Integration Reliability + Benchmark Content Precision

## Summary

LaraSkills now has a verifiable skill/knowledge interface for assistants, plus more precise Benchmark 2 architecture guidance.

## Why This Release Matters

**Before:**
Skills existed as files, but assistant access could depend on manual file browsing.

**After:**
LaraSkills generates a skill registry, exposes skill/knowledge MCP tools, and doctor can verify assistant access through the intended interface.

## What Changed

### Phase 35 — Assistant Integration Reliability

- **Skill registry**: New `.laraskills/skill-registry.json` generated during `init` and `update`, providing a verifiable skill/knowledge interface for assistant tooling.
- **6 new MCP tools** (11 total, up from 5):
  - `laraskills_list_skills` — List all installed skills with descriptions and tags
  - `laraskills_search_skills` — Search skills by name, description, or tag
  - `laraskills_read_skill` — Read the full content of a specific skill
  - `laraskills_search_knowledge` — Search the knowledge layer
  - `laraskills_retrieve_context` — Get task-focused context bundles
  - `laraskills_explain_decision` — Get architectural decision guidance with rules and anti-patterns
- **Assistant-specific doctor checks**: `laraskills doctor` validates registry integrity, MCP accessibility, and skill file access paths for OpenCode, Cursor, Claude Code, Codex, and Generic MCP.
- **Benchmark pre-flight**: `laraskills doctor --benchmark` verifies that Benchmark 2 content improvements are accessible through retrieval.
- **Doctor overhaul**: Verification now covers registry/MCP/retrieval instead of only file existence.

### Phase 36 — Benchmark 2 Content Precision

- **Calibrated architecture language**: Reduced overabsolute language in 21 knowledge files — replaced "always", "never", "every" with "prefer", "default to", "usually" except where security or data-integrity requires absolutes.
- **Cashier corrections**: Fixed imprecise proration wording and removed unsupported `Cashier::fake()` claim.
- **Queued model serialization**: Corrected `SerializesModels` description — it stores model identifiers, not full model state. Added nuanced guidance for billing vs notification jobs.
- **Webhook pipeline**: Added canonical guidance on when webhook processing should be sync vs async.
- **Spatie team authorization**: Added `viewAny` team scope guidance and API token vs team role authorization distinction.
- **Pest labeling**: Labeled unverified `toHaveTests()` syntax as conceptual pseudo-code.
- **Output mode guidance**: Added public vs internal mode guidance for architecture proposals.
- **Scoring consistency**: Added scoring consistency rule to benchmark evaluation criteria.

## Upgrade Instructions

### Global users

```powershell
npm update -g laraskills
laraskills -v
cd my-laravel-project
laraskills update --assistants all --yes
laraskills doctor --benchmark
```

### Local/project users

```powershell
npm install --save-dev laraskills@latest
npx laraskills -v
npx laraskills update --assistants all --yes
npx laraskills doctor --benchmark
```

### New users

```powershell
npm install -g laraskills
cd my-laravel-project
laraskills init --assistants all --integration full --profile core --yes
laraskills doctor --benchmark
```
