# LaraSkills 1.0.0-beta.17

**Release date:** 2026-06-22

## Summary

Beta.17 introduces `laraskills init` as the recommended project setup command,
adds `laraskills -v` / `laraskills --version`, reworks the CLI help into
clearer sections with command-specific help pages, and clarifies the hybrid
model in the README. Regression tests increased to 213/213 passing.

## What Changed

### `laraskills init` — New Recommended Project Setup

`laraskills init` replaces `laraskills install` as the recommended way to
prepare a Laravel project for AI-assisted development. It supports three
profiles (`minimal`, `core`, `full`) and installs skills, agents, rules, hooks,
MCP configs, and a `.laraskills-state.json` file.

`laraskills install` is kept working for backward compatibility.

### CLI Improvements

- **`laraskills -v` and `laraskills --version`** — Quick version display.
- **Reworked root help** — Commands organized into clear sections.
- **Command-specific help** — Each command now supports `<command> --help`
  with usage, options, and examples.

### Hybrid Model Clarification

The README now clearly explains the three layers:

1. **npm package** = CLI/MCP adapter (lightweight, always installed)
2. **full LaraSkills checkout** = knowledge source (cloned once, referenced via
   `LARASKILLS_ROOT` or `--laraskills-root`)
3. **Project files** = installed into each Laravel app with `laraskills init`
   (teaches AI agents Laravel 13 conventions)

The distinction between updating the npm package (`npm update -g laraskills`)
and refreshing project files (`laraskills update`) is now explicit.

### README Rewrite

The README was rewritten with a clean command flow: install → setup → init →
retrieve → update. Global vs local install guidance is included.

### Regression Tests

Phase 23 added CLI regression tests covering init, install, version flags, help
pages, and the hybrid model workflow. Total test count: 213 (up from 201).

## Why `init` Matters

The `init` command is the on-ramp for new users. It provides:

- Profile-based installation (`minimal` for quick start, `core` for full teams,
  `full` for maximum AI-tool compatibility)
- Explicit state tracking via `.laraskills-state.json`
- Clear separation between package installation (`npm install -g laraskills`)
  and project preparation (`laraskills init`)

## Upgrade Instructions

### Global users

```powershell
npm update -g laraskills
laraskills -v
cd my-laravel-project
laraskills update
```

### Local/project users

```powershell
npm install --save-dev laraskills@latest
npx laraskills -v
npx laraskills update
```

### New users

```powershell
npm install -g laraskills
laraskills setup --laraskills-root "C:\path\to\laraskills"
cd my-laravel-project
laraskills init
```

## Known Limitations

- The npm package is **not published** by this preparation. Publish separately.
- Knowledge-layer content is not bundled in the npm package — retrieval requires
  a full Git checkout.
- The cache is in-process only; standalone CLI invocations start separate
  processes and do not share memory.

## Publishing Instructions

Publishing to npm is a **manual** process and is not performed by this
preparation agent. To publish:

```bash
# 1. Verify the package
npm pack --dry-run

# 2. Publish with beta tag
npm publish --tag beta

# 3. Verify the published package
npm view laraskills@beta
```
