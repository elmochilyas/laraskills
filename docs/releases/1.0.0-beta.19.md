# LaraSkills 1.0.0-beta.19

**Release date:** 2026-06-22

## Summary

Beta.19 refines the `laraskills init` onboarding wizard with multi-assistant
selection, clearer MCP model language, honest support levels, and improved
doctor diagnostics. The wizard no longer mixes different concepts in one menu.

## What Changed

### Refined Init Wizard

The wizard now guides users through three clean steps:

1. **Choose coding assistants** — Select multiple assistants with comma
   (`1,2,3`), by name (`opencode,codex`), `all`, or `none`. Each assistant
   shows its support level.
2. **Choose integration level** — Full (MCP + project files), MCP only, or
   Project files only.
3. **Choose LaraSkills profile** — Core, Minimal, or Full.

A review screen confirms the plan before any files are written.

### Multi-Assistant Selection

Users can now configure multiple coding assistants in one init run:

```powershell
laraskills init --assistants all --integration full --profile core --yes
laraskills init --assistants opencode,codex --integration full --profile core --yes
laraskills init --assistants none --integration project-files --profile minimal --yes
```

The new `--assistants` flag is the preferred way to select assistants.
`--assistant`, `--tools`, and `--tool` remain as backward-compatible aliases.

### Honest Support Levels

| Assistant | Support | What LaraSkills creates |
|---|---|---|
| OpenCode | `configured` | `.opencode/opencode.json` + root `opencode.json` with MCP |
| Generic MCP | `configured` | `mcp-configs/mcp-servers.json` |
| Codex | `template` | `.codex/instructions.md` + MCP instructions |
| Cursor | `template` | `.cursor/rules.mdc` + MCP instructions |
| Claude Code | `template` | `.claude/settings.json` + MCP instructions |

Templates are useful configs and instructions, not pretend-full-setup.

### MCP Model Clarified

LaraSkills exposes one MCP server command (`laraskills-mcp`). Each coding
assistant connects to it as an MCP client. LaraSkills generates the correct
MCP client config for each assistant, or a clear template/manual instruction
when automatic wiring is not yet supported.

### Improved Doctor

`laraskills doctor` now reports per-assistant status with support levels:

```
Tool integrations:
  OpenCode: configured (auto-setup)
  Generic MCP config: configured (auto-setup)
  Codex CLI: template generated (template)
  Claude Code: template generated (template)
  Cursor IDE: template generated (template)
```

### State File

`.laraskills-state.json` now records `assistants` and `integration` fields
alongside `profile` and `tools` for accurate state tracking.

## Upgrade Instructions

Global users:

```powershell
npm update -g laraskills
laraskills -v
cd my-laravel-project
laraskills update
```

Local/project users:

```powershell
npm install --save-dev laraskills@latest
npx laraskills -v
npx laraskills update
```
