# Laravel ECC 1.0.0-beta.14

## Summary

Beta.14 introduces an in-process retrieval caching layer that significantly reduces warm-call latency for repeated retrieval, search, validate, and knowledge-unit lookups. The cache uses fingerprint-based invalidation from source-file metadata and is regression-tested for edge cases including same-size content rewrites and rapid consecutive file changes.

## Highlights

- In-process retrieval cache with 57-97% warm-latency improvement
- Fingerprint-based invalidation (mtime + file size, no persistent disk storage)
- Regression coverage for same-size rewrites and rapid consecutive rewrites
- CLI Markdown search tables with copy-friendly canonical IDs
- Working `--json` shorthand on all CLI commands
- `npm run benchmark:performance` for repeatable local measurements
- MCP round-trip verification: 10/10 checks pass
- 190 tests pass, 72/72 retrieval benchmarks pass

## In-Process Retrieval Cache

Parsed retrieval data (knowledge units, dependencies, relationships, rules, skills, checklists, anti-patterns, decision trees, aliases, external concepts) is cached in an in-process Map after the first load. All consumers benefit transparently:

- CLI commands (`retrieve`, `search`, `get`, `validate`)
- MCP tools (`retrieve_context_bundle`, `search_ecc`, `get_knowledge_unit`, `get_graph_context`, `validate_ecc`)

No code changes are needed in MCP handlers or CLI entry points — the cache is integrated at the `catalog-loader.mjs` level.

## Cache Invalidation

The cache fingerprint combines the mtime (modification time) and byte size of all 10 intelligence JSON files. The fingerprint is recomputed on every cache check (~10 quick `statSync` calls, negligible cost):

- **Normal file changes**: mtime changes — cache invalidates
- **Same-size rewrites**: different content at identical byte size — mtime changes on NTFS/APFS/ext4/XFS — cache invalidates
- **Rapid consecutive rewrites**: each write is detected independently — cache invalidates per write

**Limitation**: Coarse filesystem timestamps (VFAT/exFAT, network drives with second-level precision) may miss same-size rapid rewrites. NTFS, APFS, ext4, and XFS all provide sub-second mtime granularity adequate for beta.14.

## Performance Improvements

Warm repeated-call latency improvements (measured on Node.js v24.15.0, Windows 11, AMD Ryzen 7):

| Operation | Warm (cached) | Improvement |
|-----------|--------------|-------------|
| retrieve (standard) | ~37–104 ms | ~57%+ faster |
| search | ~23–38 ms | ~74%+ faster |
| validate | ~9.6 ms | ~92% faster |
| get knowledge unit | ~2.6 ms | ~97% faster |

**Note**: Cache is in-process only. Separate CLI invocations start separate Node.js processes and do not share memory. Long-lived MCP processes benefit most from caching.

## CLI Output Improvements

- **Markdown search tables**: Search output now uses readable Markdown tables with canonical IDs as the first column, making IDs copy-paste friendly for subsequent `get` or `get_graph_context` calls
- **`--json` shorthand**: The `--json` flag now works correctly on all CLI commands (`retrieve`, `search`, `get`, `validate`)
- **Deterministic output**: Retrieval output remains deterministic and backward-compatible

## Package Distribution Strategy

The npm package intentionally excludes the full knowledge and intelligence layers. Retrieval requires access to a full Laravel ECC checkout. Run `laravel-ecc setup` to configure the checkout path.

No remote MCP hosting exists. The MCP server (`laravel-ecc-mcp`) runs locally and communicates via standard I/O.

## Known Limitations

- Cache is in-process only — separate CLI invocations start new processes and do not share state
- Cold-start latency is unchanged (~160-270 ms for operations that require loading all 10 JSON files)
- Coarse filesystem timestamps (second-level precision) may not detect same-size rapid rewrites on VFAT/exFAT or some network drives
- No persistent disk cache — intentional, as cold-start time is the dominant factor

## Upgrade Notes

Beta.14 is backward-compatible with beta.13. No configuration changes are required.

### New scripts

```bash
# Run performance benchmarks
npm run benchmark:performance
```

### CLI changes

```bash
# Search with Markdown tables and --json shorthand
laravel-ecc search "your query" --limit 5
laravel-ecc search "your query" --limit 5 --json

# All other commands remain unchanged
laravel-ecc retrieve "your task"
laravel-ecc validate
laravel-ecc doctor
```

### Updating

```bash
npm install -g laravel-ecc@beta
# or if installed locally
npm install laravel-ecc@beta
```
