# Laravel ECC 1.0.0-beta.12

## Summary

Phase 12 delivers cross-platform onboarding improvements: a shared CLI + MCP ECC root resolver, persisted user configuration, `laravel-ecc setup` and `laravel-ecc doctor` commands, actionable diagnostics, updated documentation, and 24 new automated tests.

## Highlights

- **Shared CLI + MCP ECC root resolver** — One deterministic resolver shared between the CLI and MCP adapter, eliminating duplicated path-discovery logic
- **Deterministic root-resolution precedence** — Clear, predictable order: `--ecc-root` flag > `ECC_ROOT` environment variable > persisted user config > cwd and parent-directory discovery > actionable failure
- **Persisted cross-platform user configuration** — Windows: `%APPDATA%\laravel-ecc\config.json`; Linux/macOS: `$XDG_CONFIG_HOME/laravel-ecc/config.json` or `~/.config/laravel-ecc/config.json`
- **`laravel-ecc setup` command** — Configure the ECC root path interactively or via `--ecc-root "<path>"`
- **`laravel-ecc doctor` command** — Comprehensive health check with actionable diagnostics when issues are detected
- **Actionable MCP and CLI errors** — Error messages now guide the user toward resolution instead of showing raw exceptions
- **Updated onboarding documentation** — Clearer setup instructions covering installation, configuration, and verification
- **24 new automated tests** — Covering root resolver, user config, setup command, doctor command, and edge cases

## Setup Command

```bash
# Interactive setup
npx laravel-ecc setup

# Specify root directly
npx laravel-ecc setup --ecc-root "/path/to/laravel-ecc"

# Verify after setup
npx laravel-ecc doctor
```

The setup command persists the ECC checkout path to the user-level configuration file, enabling all subsequent CLI and MCP operations to locate the knowledge layer.

## Doctor Command

```bash
npx laravel-ecc doctor
```

The doctor command validates:

- Node.js version (18+ required)
- ECC root path resolution (evaluates all precedence levels)
- Knowledge layer integrity (KU count, edge counts)
- Configuration file validity
- MCP adapter accessibility

Reports a HEALTHY / UNHEALTHY status with specific, actionable guidance for each failing check.

## Shared Root Resolver

Located in `src/runtime/ecc-root-resolver.mjs`, the shared resolver implements five-level precedence:

1. `--ecc-root` CLI argument
2. `ECC_ROOT` environment variable
3. Persisted user config (`config.json`)
4. cwd and parent-directory discovery (looks for `knowledge/` and `intelligence/` directories)
5. Actionable failure with specific guidance

## Persisted Configuration

Cross-platform user configuration stored at:

- **Windows:** `%APPDATA%\laravel-ecc\config.json`
- **Linux/macOS:** `$XDG_CONFIG_HOME/laravel-ecc/config.json` or `~/.config/laravel-ecc/config.json`

```json
{
  "eccRoot": "C:\\Users\\me\\laravel-ecc",
  "configuredAt": "2026-06-10T12:00:00.000Z"
}
```

## MCP Integration

The MCP adapter (`scripts/laravel-ecc-mcp.mjs`) now shares the same root resolver as the CLI, ensuring consistent behavior across both interfaces. All five MCP tools remain unchanged:

- `retrieve_context_bundle`
- `search_ecc`
- `get_knowledge_unit`
- `get_graph_context`
- `validate_ecc`

## Package Distribution Strategy

The npm package intentionally excludes the full knowledge and intelligence layers. Retrieval requires access to a full Laravel ECC checkout. Run `laravel-ecc setup` to configure the checkout path.

**Included in npm package:**

- `scripts/laravel-ecc.mjs` — CLI entry point
- `scripts/laravel-ecc-mcp.mjs` — MCP adapter
- `scripts/mcp/` — MCP tool handlers and schemas
- `src/runtime/` — Shared runtime modules (root resolver, user config)
- `skills/`, `rules/`, `agents/`, `commands/` — Operating layer
- Install scripts, config templates, documentation

**Excluded from npm package:**

- `knowledge/` — 2,321 knowledge units
- `intelligence/` — Indexes and JSON intelligence
- `tools/` — Rebuild and generation scripts
- `tests/` — Test suites
- `examples/` — Example configurations

## Known Limitations

- `laravel-ecc setup --clone` is not included yet — users must clone the repository manually before running setup
- Remote MCP hosting is not available — the MCP adapter runs locally
- Full knowledge layer retrieval requires a local ECC checkout configured via `setup`

## Upgrade Notes

From `1.0.0-beta.11`:

1. Update the npm package: `npm install laravel-ecc@beta`
2. Run `npx laravel-ecc setup` to configure your ECC root path
3. Run `npx laravel-ecc doctor` to verify the installation
4. Existing CLI workflows remain unchanged
5. MCP configurations remain unchanged — the shared resolver automatically uses the same root
