# LaraSkills 1.0.0-beta.16

## Summary

Beta.16 is the beta release preparation for LaraSkills, incorporating Phase 17
real Laravel validation (fresh project, existing project, MCP workflow) and
Phase 18 release readiness polish (LICENSE, README cleanup, package metadata).
This release validates LaraSkills in real Laravel 13 usage scenarios and fixes
two critical retrieval defects discovered during testing.

## Phase 17 — Real Laravel Usage Validation

### Fresh Laravel Project

LaraSkills was installed from a packed tarball into a new `laravel/laravel`
v13.0 project. The CLI and `full` profile installed cleanly, `doctor` reported
HEALTHY, and `validate` confirmed a valid intelligence graph (2,321 KUs,
427 dependency edges, 3,513 relationship edges, zero defects).

A coding agent used Compact and Standard retrieval + targeted search to build
a complete products CRUD API with Sanctum authentication, Form Requests,
ProductPolicy, API Resources, cursor pagination, rate limiting, and Pest tests.
The generated application finished with 11 tests and 168 assertions. No
unpublished files were required outside the declared LaraSkills root.

**Result: PASS**

### Existing Laravel Project

LaraSkills was introduced to a non-empty Breeze/Pest application without
overwriting existing files. The project had 29 baseline tests, 73 assertions,
migrations, Blade views, routes, Pint passing, and a Vite production build.

The one-task maintenance action added note archiving with policy authorization.
The regression test failed before the fix (expected RED) and passed after.
The final application had 32 tests and 82 assertions. All Laravel package
gates (test, Pint, migration, route list, Composer audit, production npm audit)
passed. No pre-existing tests regressed and no unrelated application file was
overwritten.

**Result: PASS** (with environment limitation: coverage driver unavailable)

### OpenCode MCP Workflow

OpenCode launched `laraskills-mcp` from the packed package, discovered all
five read-only tools, and completed a signed webhook implementation (HMAC
SHA-256 verification, timing-safe comparison, replay protection, queued job
dispatch, Pest tests) using only MCP tools — no CLI or direct file fallback.
The canonical-ID round trip through `search_ecc`, `get_knowledge_unit`, and
`get_graph_context` succeeded.

**Result: PASS**

## Defects Found and Fixed

### 1. Irrelevant Retrieval Ranking (Critical)

Compact and Standard retrieval ranked
`data-storage-systems/replication/laravel-read-write-config` first for a
policy-authorization task. Root cause: two invalid ranking signals — a
multi-token alias could match from one shared token, and every KU with any
alias received an unconditional `aliasTarget` bonus. Both signals were removed.

### 2. Missing `get --include-content` Body (High)

The `--include-content` flag returned metadata only, omitting the actual
`04-standardized-knowledge.md` document body. The formatter now resolves the
content path beneath the selected LaraSkills root and appends the full body.

## Known Limitations

- The npm package is **not published** by this preparation. Publish separately.
- Knowledge-layer content is not bundled in the npm package — retrieval requires
  a full Git checkout.
- Coverage driver was unavailable in the existing-project PHP environment.
- VFAT/exFAT filesystems may not detect same-size rapid file rewrites reliably.
- The cache is in-process only; standalone CLI invocations start separate
  processes and do not share memory.

## Upgrade Notes

Install the beta tag:

```bash
npm install --save-dev laraskills@beta
npx laraskills install --profile core
npx laraskills setup --laraskills-root /path/to/laraskills
npx laraskills doctor
```

If migrating from `laravel-ecc`, update binary names and environment variables.
Temporary `laravel-ecc` / `laravel-ecc-mcp` / `ECC_ROOT` compatibility aliases
are retained but will be removed in a future release.
