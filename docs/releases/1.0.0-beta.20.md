# LaraSkills 1.0.0-beta.20

**Release date:** 2026-06-22

## Summary

Beta.20 delivers automatic MCP configuration for all supported coding assistants.
`laraskills init --assistants all --integration full --profile core --yes` now
writes the correct MCP client config for OpenCode, Claude Code, Cursor, Codex,
and Generic MCP — with safe merge, existing server preservation, and backups.

## What Changed

### Automatic MCP Configuration

Every assistant now gets its native MCP config written automatically:

| Assistant | Config file | Status |
|---|---|---|
| OpenCode | `.opencode/opencode.json` + `opencode.json` | configured |
| Claude Code | `.mcp.json` | configured |
| Cursor | `.cursor/mcp.json` | configured |
| Codex | `.codex/config.toml` | configured |
| Generic MCP | `mcp-configs/laraskills-mcp.json` | configured |

No more "template-only" or "manual step required" — every assistant is
configured automatically.

### Safe Merge & Backup Behavior

- **Existing MCP servers are preserved**. If you already have a `github` or
  `other-server` MCP entry in your config, LaraSkills appends alongside it.
- **Only the `laraskills` entry is touched**. Unrelated settings and keys are
  never modified.
- **Backups are created** before any config file is changed
  (`filename.backup-<timestamp>`).
- **Init is idempotent**. Running `laraskills init` on an already-configured
  project skips files that already have the LaraSkills MCP entry.
- **`--dry-run` writes nothing**.

### Wizard & Doctor Updates

The `laraskills init` wizard now labels all assistants consistently since all
are configured automatically. `laraskills doctor` reports `configured` for
assistants whose MCP config file exists and contains the LaraSkills server
entry.

## Why MCP Auto-Configuration Matters

LaraSkills exposes one MCP server (`laraskills-mcp`). Each coding assistant is
an MCP client with its own config format:

- OpenCode uses a project-level `opencode.json` with `"mcp"` key
- Claude Code uses a project-level `.mcp.json` with `"mcpServers"` key
- Cursor uses `.cursor/mcp.json` with `"mcpServers"` key
- Codex uses `.codex/config.toml` with TOML `[mcp_servers]` tables
- Generic MCP uses `mcp-configs/laraskills-mcp.json`

LaraSkills now writes the correct format for each tool. No manual wiring.

## Upgrade Instructions

Global users:

```powershell
npm update -g laraskills
laraskills -v
cd my-laravel-project
laraskills update --assistants all --yes
```

Local/project users:

```powershell
npm install --save-dev laraskills@latest
npx laraskills -v
npx laraskills update --assistants all --yes
```

New users:

```powershell
npm install -g laraskills
cd my-laravel-project
laraskills init --assistants all --integration full --profile core --yes
```
