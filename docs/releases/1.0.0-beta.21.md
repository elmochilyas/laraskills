# LaraSkills 1.0.0-beta.21 (Hotfix)

**Release date:** 2026-06-22
**Type:** Hotfix release

## Summary

Beta.21 is a hotfix for a bug in beta.20 where LaraSkills generated `.opencode/opencode.json` with `{file:commands/plan.md}` (and other command file) references but did not copy the referenced `.opencode/commands/` directory. OpenCode rejected the generated configuration with a missing file error.

## Root Cause

```
LaraSkills copied `.opencode/opencode.json` into the target project but did
not copy the `.opencode/commands/` directory. Since the config referenced
files like `{file:commands/plan.md}`, OpenCode rejected the generated
configuration.
```

## What Changed

### Fixes

- **OpenCode config**: `opencode.setup()` now copies `.opencode/commands/` alongside `.opencode/opencode.json`. All four referenced command files (plan, tdd, code-review, artisan) are guaranteed to exist.
- **Harness configs**: `copyHarnessConfigs()` now copies `commands/` subdirectories for any harness dir that has one.
- **Doctor validation**: `validateOpenCodeFileReferences()` recursively scans `.opencode/opencode.json` for `{file:...}` references and verifies each resolved path exists.
- **Doctor output**: When OpenCode is selected and file references are broken, `laraskills doctor` reports `OpenCode: broken` with the specific missing files and a repair command.
- **Update repair**: `laraskills update --assistants opencode --yes` repairs missing command files created by a beta.20 init, without overwriting unrelated user config.
- **Install summary**: The init summary now shows only user-selected assistants (e.g., "OpenCode"). Auto-generated shared MCP config appears as a separate "Shared MCP config: generated" line.

### Preserved Behavior

- Existing OpenCode config (`opencode.json`) with custom MCP servers remains untouched.
- Existing `.opencode/opencode.json` user settings are deep-merged safely.
- `--dry-run` mode works correctly.

### Tests

- 15 new regression tests covering config generation, update repair, doctor detection, config preservation, idempotency, and summary accuracy.
- Total test suite: 244 tests, 0 failures.

## Why This Hotfix Matters

Anyone who ran `laraskills init --assistants opencode` on beta.20 encountered this error when launching OpenCode:

```text
Configuration is invalid at .opencode/opencode.json:
bad file reference: "{file:commands/plan.md}"
.opencode/commands/plan.md does not exist
```

Beta.21 resolves this silently for new installs and provides a one-command repair for existing beta.20 installs.

## Upgrade Instructions

### Global users

```powershell
npm update -g laraskills
laraskills -v
cd my-laravel-project
laraskills update --assistants opencode --yes
laraskills doctor
```

### Local/project users

```powershell
npm install --save-dev laraskills@latest
npx laraskills -v
npx laraskills update --assistants opencode --yes
npx laraskills doctor
```

### Clean reinstall

```powershell
npm uninstall -g laraskills
npm install -g laraskills@latest
laraskills -v
```
