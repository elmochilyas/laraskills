# LaraSkills 1.0.0-beta.18

**Release date:** 2026-06-22

## Summary

Beta.18 delivers zero-friction onboarding. Normal users can now install
LaraSkills from npm, run `laraskills init` inside a Laravel project, and use
packaged intelligence without ever cloning the LaraSkills repository.
`doctor`, `validate`, `retrieve`, `search`, `get --include-content`, and MCP
all work out of the box.

## Main User-Facing Change

**No manual clone required.** The npm package now bundles all 2,321 knowledge
units' intelligence JSON files plus a compact standardized content index,
making the full retrieval pipeline work without `LARASKILLS_ROOT`,
`laraskills setup`, or a Git clone.

## New Onboarding Flow

```powershell
npm install -g laraskills
cd my-laravel-project
laraskills init
laraskills doctor
laraskills retrieve "Add a Laravel policy and Pest tests"
```

## What Changed

### Packaged Intelligence

- `intelligence/json/` (10 JSON files: KUs, dependencies, relationships, rules,
  skills, checklists, anti-patterns, decision trees, aliases, external concepts)
  now shipped in the npm package.
- `intelligence/content/content-index.json` maps all 2,321 canonical KU IDs to
  their full standardized Markdown content. `get --include-content` returns
  real knowledge bodies — not metadata-only placeholders.
- Root resolution falls back to packaged intelligence before throwing errors.

### Interactive `laraskills init`

- Detects whether the current directory is a Laravel project.
- Guides the user through profile selection (core/minimal/full) and tool
  integration selection (OpenCode, Generic MCP, Codex, Claude Code, Cursor).
- Non-interactive mode available for CI/CD:
  ```
  laraskills init --profile core --tools opencode --yes
  ```

### Tool Integration Framework

- **OpenCode** (fully supported): Creates/merges `.opencode/opencode.json` and
  root `opencode.json`. Sets up instructions, sub-agents (planner, code-reviewer,
  security-reviewer, tdd-guide, build-error-resolver), slash commands, and MCP
  connection. Existing user config is preserved and backed up.
- **Generic MCP** (fully supported): Creates `mcp-configs/mcp-servers.json`.
- **Codex, Claude Code, Cursor** (template): Generates instructions template
  files. Manual MCP wiring required.

### Improved `doctor`

Reports two sections:
- **Machine**: Intelligence source type (packaged/configured), retrieval
  readiness, MCP adapter availability.
- **Project**: Laravel detection, LaraSkills initialized state, profile,
  installed components, tool integration status (OpenCode configured, etc.).

### Other Changes

- `laraskills setup` is now optional/advanced — normal users don't need it.
- `laraskills update` supports `--tools` and `--dry-run`.
- Root resolution priority: `--laraskills-root` → `LARASKILLS_ROOT` → saved
  config → packaged intelligence → legacy fallback.
- MCP error messages suggest package reinstall rather than manual clone.
- Windows-safe postinstall message via dedicated script file.
- Backward compatibility preserved for `install`, `setup`, `--ecc-root`,
  `ECC_ROOT`, `laravel-ecc`, and existing project updates.

## Upgrade Instructions

Global users:

```powershell
npm update -g laraskills
laraskills -v
cd my-laravel-project
laraskills update
```

Local/project users:

```powershell
npm install --save-dev laraskills@latest
npx laraskills -v
npx laraskills update
```

Clean install test:

```powershell
npm install -g laraskills
cd my-laravel-project
laraskills init
```

## Package Size

| Metric | Value |
|--------|-------|
| Files | 152 |
| Compressed | 4.8 MB |
| Unpacked | 32.1 MB |

The size increase from beta.17 (150 files, 821 kB / 15.4 MB) is due to the
bundled intelligence JSON files (10 files, ~15 MB) and the standardized
content index (1 file, ~16 MB).

## Known Limitations

- `get --include-content` returns the full standardized knowledge body from the
  packaged content index. The six phase files (05-rules.md through
  09-checklists.md) still require a full Git checkout.
- Template-only tool integrations (Codex, Claude Code, Cursor) generate
  instruction files but require manual MCP wiring.
