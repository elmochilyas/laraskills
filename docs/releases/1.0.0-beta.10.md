# Laravel ECC 1.0.0-beta.10

## Summary

Authorization enforcement retrieval hardening, MCP startup reliability fixes, and real-project post-hardening verification.

## Highlights

- Authorization enforcement guidance strengthened across knowledge units
- Authentication vs authorization separation clarified in targeted authorization guidance
- Registered-but-unused-policy anti-pattern documented and regression-tested
- Endpoint-level policy enforcement checklist added
- 401 vs 403 test guidance added to the authorization-testing guidance
- `sometimes|required` update-validation guidance added for partial updates
- Authorization retrieval benchmark bench-071 added
- MCP canonical-ID round-trip regression coverage added
- MCP unknown-ID errors improved with clearer messaging
- MCP stderr startup race fixed for reliable MCP Inspector operation
- Full test suite increased to 150 tests
- Benchmarks increased to 71 tasks

## Authorization Enforcement Hardening

- Every protected endpoint must explicitly execute authorization
- Policy registration alone is insufficient — policies must be invoked
- Authentication is not authorization; both must be tested separately
- Test both 401 (unauthenticated) and 403 (authenticated-but-unauthorized) behavior
- Added `registered-but-unused-policy` anti-pattern detection
- Added `sometimes|required` guidance for update fields that may be omitted but must not be empty when present

## MCP Usability Improvements

- Fixed MCP stderr startup-order race that caused intermittent failures with MCP Inspector
- Added search_ecc → canonical ID → get_knowledge_unit regression coverage to ensure round-trip integrity
- Improved error messages for unknown IDs passed to get_knowledge_unit

## Retrieval and Benchmark Improvements

- Added bench-071 for authorization enforcement retrieval quality
- Full retrieval benchmark suite: 71 / 71 tasks passing at 100%

## Real-Project Verification

Phase 11.5 repeated the Product CRUD experiment from the same clean Laravel baseline after the Phase 11.4 retrieval hardening. The ECC-assisted implementation now included explicit endpoint-level policy checks for create, update, and delete operations. This controlled repeat provides strong evidence that the remediation improved the generated implementation, while remaining a single integration experiment.

## Validation Results

- 150 / 150 tests passing
- 71 / 71 benchmarks passing at 100%
- 2,321 knowledge units intact
- 429 dependency edges, 0 cycles
- 3,513 relationship edges, 0 self-loops
- 120 aliases, 26 external concepts
- Intelligence layer VALID

## Package Distribution Strategy

The lightweight npm package intentionally excludes the full knowledge and intelligence layers.
`ECC_ROOT` must point to a full Laravel ECC checkout for retrieval and MCP use.

Remote MCP hosting is not available in this release.

## Known Limitations

- The npm package does not bundle knowledge/ or intelligence/ directories
- The lightweight npm package requires access to a full Laravel ECC checkout. Set ECC_ROOT when the checkout cannot be resolved automatically.

## Upgrade Notes

- Update `ECC_ROOT` if needed after upgrading the package
- No intentional breaking changes to the CLI or MCP API
- All five MCP tools retain their existing signatures
- Existing workflows using `npx laravel-ecc retrieve`, `search`, `get`, `validate` continue unchanged
