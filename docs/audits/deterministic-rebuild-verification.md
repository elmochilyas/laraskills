## Summary

The rebuild pipeline is now **deterministic**. Two independent runs of `inject-dependency-edges.ps1` produce identical SHA-256 hashes for all 10 intelligence JSON files.

---

## Non-Determinism Fix

### Root Cause

The old `inject-dependency-edges.ps1` script iterated over PowerShell hash-table keys in insertion order (non-deterministic):

```powershell
# OLD â€” unstable
foreach ($kuId in $explicitRelated.Keys) { ... }
```

When a reference string matched multiple KUs via fuzzy substring match, the first match won (`break` on first hit). Changing the order in which keys were iterated changed which KU matched for thousands of references.

### Fix Applied

All key iterations are now sorted:

```powershell
# NEW â€” deterministic
foreach ($kuId in ($explicitRelated.Keys | Sort-Object)) { ... }
```

Additionally added guard against relationship self-loops:

```powershell
# NEW â€” self-loop rejection
if ($targetId -ne $kuId) { ... }
```
